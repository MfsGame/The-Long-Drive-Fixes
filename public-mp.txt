# TLD Public MP — plugin manifest (v2.3 known-good)
#
# Format: <PluginAssemblyName>  <Version>  <DownloadURL>
# Lines starting with # are comments. Blank lines are skipped.
#
# The TLDPubMPUpdater patcher fetches this file each launch, compares
# <Version> against the BepInPlugin version of each local DLL, and downloads
# anything newer as *.dll.update (applied on next launch).
#
# v2.3 (2026-05-27): adds two physics improvements on top of the v2.2
# conservative-fixes baseline.
#
#   - TLDPubCarSync 0.1.0
#       Bumps the host's per-car position broadcast from stock 1Hz to a
#       configurable rate (default 20Hz). Narrows the gap during which the
#       receiving client's local sim can diverge from authoritative state from
#       ~1000ms to ~50ms. Idle-skips parked vehicles so bandwidth scales with
#       moving cars only.
#
#   - TLDPubRemoteCarKinematic 0.1.0
#       Sets isKinematic=true on the rigidbody of any car the local side
#       isn't currently driving (host-driven cars on the client, etc.). Stops
#       the wheel-rigidbody-vs-network-position tug-of-war that caused the
#       up/down "bouncing like overloaded suspension" symptom under any
#       network latency. Local physics resumes as soon as the local player
#       enters the car.
#
# Also included (off by default, opt-in via [Testing] Enabled = true in their
# respective configs) for two-instance local MP testing:
#
#   - TLDPubLoopback 0.15.0       — file-bridge transport + NetSim + lobby fake
#   - TLDPubFakeId 1.0.0           — patches SteamUser.GetSteamID for ID swap
#
# These ship in /public so users can easily set up local testing, but they
# are fully inert until you explicitly turn on [Testing] Enabled = true in
# the Loopback config (which is the single master flag). Do NOT enable
# testing mode if you want to play with a real friend over real Steam P2P.
#
# This snapshot was confirmed working under NetSim 100ms±50ms latency, 4%
# packet loss in the local two-instance loopback test setup.
