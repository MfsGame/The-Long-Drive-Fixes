# TLD Public MP — plugin manifest (v2.1 known-good)
#
# Format: <PluginAssemblyName>  <Version>  <DownloadURL>
# Lines starting with # are comments. Blank lines are skipped.
#
# The TLDPubMPUpdater patcher fetches this file each launch, compares
# <Version> against the BepInPlugin version of each local DLL, and downloads
# anything newer as *.dll.update (applied on next launch).
#
# This snapshot was confirmed working by Reedo in a real-MP session +
# NetSim-emulated session (100ms±50ms latency, 4% packet loss). The Driver
# Authority + Fluid Dedupe + Player Stable + Body Push combination
# specifically addressed:
#   - "Rev but no movement" while driving (DriverAuthority — stops echoed
#     stale inputs from clobbering the driver's W press under latency)
#   - Friend disappears from seat and re-pops (PlayerStable — extends the
#     stock 1.5s player-destroy timeout to 5s so brief network/GC stutters
#     don't destroy avatars)
#   - Pushed items snap back (BodyPush — broadcasts pushablescript.PushLocal
#     impulses that stock TLD silently drops)
#   - Bandwidth saturation (FluidDedupe — kills ~95% of the 32k fluid-state
#     packets/5min spam that stock TLD broadcasts; measured directly off the
#     bridge files under load).
#
# All four are pure additive Harmony patches. No claim/authority changes, no
# transport replacement, no smoothing. The previous experimental versions
# that broke things (ClaimDrivenCars, ClaimMovingItems, SteamMatchmaking
# fake-lobby, ItemSmooth) are NOT in this manifest and have been removed
# from /public — their sources remain in /src for future revisit.
#
# Dev-only tools (TLDPubLoopback v0.12 with NetSim, TLDPubBridge HTTP
# introspect) are also NOT in this manifest. They live in /src and can be
# manually installed for local two-instance testing if you want to
# reproduce real-network conditions without your friend online.
